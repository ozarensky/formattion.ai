Who we are

The name
for · matt · ion

The company is founded by Matt and Ion. The name “Formattion” reflects the three core components of the business: for (the internal AI entity), matt, and ion.

The mission
Closing the automation gap between large construction firms and the subcontractors who build Britain.

UK subcontract businesses operate with fragmented systems and heavy manual workload. Large firms automate. SMEs are left behind. formattion.ai changes that.

Two distinct modes
Light mode is warm cream — organic, approachable, human. Dark mode is cool neutral — precise, authoritative, focused. They are complementary, not mirror images. The warm accent threads both.
The shared accent
The blue-grey accent (#4A6278 in light, #6E8299 in dark) is the visual constant across both modes. It shifts in value to maintain contrast, but its character never changes.
Why warm light, cool dark
The light mode speaks to the client — warm, human, trustworthy. The dark mode speaks to the operator — precise, tool-like, focused. Both are correct for their context.
Mode context
Light mode for client-facing outputs: reports, emails, proposals, the marketing site. Dark mode for the audit platform UI, the For agent interface, and any high-focus operational context.

Corner rounding is functional, not decorative. Each radius value maps to a specific role in the UI. Do not mix values arbitrarily — consistent radius use signals hierarchy and builds spatial familiarity across the platform.

Token Value Usage
--rx-tight 6px Text inputs, select fields, inline tags, code badges, table cells with interactive state
--rx-std 12px Standard cards, panels, modals, section containers, audit form blocks, nav drawer
--rx-feature 20px Hero blocks, feature callouts, full-bleed image containers, report cover panels
--rx-pill 999px CTA buttons, status badges, filter chips, eyebrow pills, toggle tracks
do
Use rx-std (12px) as the default for any contained element
Match child radius to parent — never exceed the container's value
Apply rx-pill exclusively to fully-rounded interactive elements
Use rx-tight consistently across all form inputs in a single context
don't
Mix radius values within a single component
Apply rx-feature to interactive controls — it reads as decorative
Use arbitrary values outside the four tokens
Over-round small elements — a 4px tag in a 20px container looks broken

Voice & tone

The brand speaks the way the methodology works — structured, direct, precise. The audience is a construction operator. They read fast and they know when something is trying too hard.

Structured
Every output has a clear architecture. Nothing placed arbitrarily. The report mirrors the methodology.
Consistent
The same standard applied every time, to every client, on every site.
Reliable
Professionals trust formattion because it delivers — without performance, without hedging.
Predictable
In construction, predictability is the premium product. The brand owns it.
Precise
Named specifics over vague summaries. Scores, dates, section numbers, £ values.
Considered
Nothing is rushed. Recommendations come from the audit, not instinct. The roadmap is phased.
do
Write in complete, purposeful sentences
Active voice — "The audit identified three gaps"
Name specifics — section numbers, scores, dates, £ values
Be direct with findings, including uncomfortable ones
Lead with the outcome, follow with the detail
Write for someone reading standing up on a job site
don't
Use filler — "It's worth noting that..."
Soften findings with hedging or apology
Use exclamation marks or informal punctuation
Passive voice — "Issues were found..."
Use tech jargon without a plain-English follow-up
Over-explain — the client runs a business
For — AI agent voice
"Section 4 shows three material storage gaps outside current site standards. The highest-risk item has been flagged for immediate review. A structured remediation path is outlined in the roadmap below."
Client report — opening
"This audit identified clear opportunities to strengthen your site management processes. The findings below are structured by priority — each with a specific, actionable recommendation and an implementation timeline."
Outreach — operator register
"Most subcontract businesses lose £30–80k a year to slow quote follow-up and missed enquiries. We can show you exactly where yours are going — and fix it in six weeks."

Image system

All images are generated via diffusion model API. The style is locked — only subject and composition variables change per use case. This ensures visual consistency across every context without a photographer or art director on every job.

Core aesthetic direction
Documentary realism, desaturated warm
Images feel like they were taken on site — not staged, not stock. Real construction environments, real materials, real work. Treated with a warm desaturated grade that echoes the brand palette. The feel is quiet authority, not glossy aspiration.
Style rules — always applied
01Desaturated warm grade. Colours muted to 20–40% saturation. Aged document feel — not black-and-white, not vivid.
02Ambient natural light. Overcast daylight, golden hour, or diffused interior. No studio flash, no dramatic spotlights.
03No faces. Workers may appear from behind, side-on, or at distance. Hands on equipment are permitted. Faces are never shown.
04Geometry and structure over chaos. Clear lines, repetition, material texture. Steel framework, concrete surfaces, organised sites, aerial groundworks.
05Intentional negative space. Leave room in the composition — text will often sit here. Do not fill the frame edge-to-edge.
06Film grain texture. 15–25% opacity grain. Makes AI-generated images feel documentary rather than synthetic.

LinkedIn

Prompt architecture — locked base + dynamic variables
Every image prompt is assembled from four layers in order. The style base is never modified. Only the subject, composition, and context spec are generated per use case by For.
[STYLE BASE] documentary construction photography, desaturated warm film grade, ambient overcast daylight, subtle grain texture, no faces, architectural precision, negative space composition —
[SUBJECT] {{generated by For from topic}} —
[COMPOSITION] {{overhead aerial / close detail / wide environmental / etc.}} —
[CONTEXT SPEC] {{aspect ratio + text zone from context template}}

--no faces, bright colours, studio lighting, stock photo style, cgi render, illustration, logo, text
Generation
Model: Flux 2 via fal.ai. Automated via n8n — POST to webhook with context, topic, and optional industry. For generates subject + composition via GPT-4o. Prompt assembled and sent to Flux 2 API. Image URL returned.
Never generate
Faces or identifiable people. Bright or saturated colour palettes. Studio-lit product shots. Stock photo aesthetics. CGI renders or illustrations. Text within the image. Any scene that reads as aspirational rather than operational.
